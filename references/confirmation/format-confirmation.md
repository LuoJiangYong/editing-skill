## 一、确认时机

L3 格式消费完成后、L4 发布建议生成前。即 output-review.html 已生成，用户在浏览器中审阅排版效果后确认。

---

## 二、确认三项

### 2.1 排版样式

**确认内容：**

- 整体风格是否符合预设格式（tech-magazine 等）的设计规范？
- 配色/字号/间距/组件是否与 design.md 基线一致？
- 是否有样式错乱（溢出/重叠/字号不统一/颜色突兀）？
- 是否适配目标平台（微信公众号后台粘贴预览）？

**检查点：**

```
□ 标题层级（h1/h2/h3）字号和间距符合设计规范
□ 正文字号适中（微信推荐 15-17px）
□ 行间距舒适（1.6-1.8em）
□ 引用块样式正确
□ 列表缩进和符号正确
□ 图片宽度适配（不超过 100% 父容器宽度）
□ 链接颜色可辨识
□ 分隔线/留白视觉舒适
□ 暗色模式/亮色模式的基础兼容性（如使用 CSS 变量）
□ 代码块（如有）可读性
```

**呈现方式：** Agent 在浏览器中打开 `output-review.html`，截图或引导用户查看。

**通过标准：** 用户确认"排版OK"或提出具体调整。无样式崩溃/错乱。

**回流规则：** 样式调整 → 回到 L3，修改格式模板参数 → 重新渲染 output-review.html。

---

### 2.2 组件选择

**确认内容（对使用了非正文组件的格式）：**

- 卡片/表格/时间线/对比表等组件是否正确渲染？
- 组件间距是否与正文段落协调？
- 组件内部内容是否完整（无截断/溢出）？

**常见组件检查：**

| 组件 | 检查项 |
|------|-------|
| 信息卡片 | 标题/描述/标签/链接 完整，阴影/圆角一致 |
| 对比表格 | 列宽合理，表头清晰，移动端可横向滚动 |
| 时间线 | 节点对齐，日期-事件对应 |
| 引用块 | 缩进/左侧竖线/来源标注 |
| 图片 | 加载正常、alt 文本、四周留白 |
| 脚注区 | 分隔线 + 逐条列出 |

**通过标准：** 所有组件渲染正常，无布局破损。

**回流规则：** 组件问题 → 调整对应组件模板 → 重新渲染。

---

### 2.3 输出规格

**确认内容：**

- 目标平台兼容性：微信公众号后台粘贴后预览是否正常？
- HTML 文件大小是否在平台限制内？（微信公众号单篇建议 < 50KB HTML）
- 图片是否使用了外部 CDN（微信不支持外部图床，需确认图片方案）？
- 是否生成了最终版 `output.html`（经确认后从 review 版固化）？

**平台兼容性速查：**

| 平台 | 关键限制 |
|------|---------|
| 微信公众号 | 不支持外部 CSS 文件（必须内联 style）、不支持 JS、不支持外部图床（需上传到微信素材库）、`<iframe>` 不显示 |
| 知乎 | 支持部分 HTML，不支持 `<style>` 标签，主要靠富文本编辑器 |
| Newsletter（Revue/Substack） | 支持内联 CSS + 有限的外部图片 |
| 小红书 | 非 HTML 格式，走 infographic-carousel 生成图片 |

**微信公众号特殊检查：**

```
□ 所有 CSS 内联到 HTML（无 <link> 外部样式表）
□ 无 <script> 标签
□ 无 <iframe>
□ 图片使用占位符或 base64（提醒用户上传后替换）
□ 无 position:fixed/sticky（微信会过滤）
□ 无 animation/transition（微信不支持）
□ section 用 <section style="...">（微信不支持 class/id 选择器）
□ 字号 ≥ 14px（微信最小可读字号）
```

**通过标准：** 目标平台兼容性通过。用户确认"可以发布"。

**回流规则：** 平台兼容性问题 → 回到 L3 调整渲染参数 → 重新输出。

## 四、快速通过条件

以下情况下格式确认可简化：

1. **用户明确跳过**："格式跳过""直接输出"。
2. **重复格式生产**：同一格式第 N 次使用，前几次已验证过模板。
3. **仅文本型输出**：无特殊组件，纯 Markdown→HTML 直转。

简化时仍需检查微信后台兼容性（如果有）。

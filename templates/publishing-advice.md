## 一、输出格式

L4 发布建议以卡片形式输出，在格式确认通过后生成。

```
══════════════════════════════════════════════
          发布建议卡
══════════════════════════════════════════════

📄 内容：<标题>
📅 建议发布日期：<YYYY-MM-DD>（<星期X>）
🕐 建议发布时间：<时间>（<理由>）

📋 标题变体：
  A. <主标题>（推荐）
  B. <疑问句变体>
  C. <数字驱动变体>
  D. <悬念式变体>

🖼️ 封面方向：
  - 主视觉：<建议>
  - 风格：<建议>
  - 备选：<建议>

📱 发布平台优先级：
  1. 微信公众号（主平台）
  2. <其他平台>

📊 传播建议：
  - 转发语建议：<1-2 句话>
  - 社群分享角度：<建议>

⚠️ 注意事项：
  - <注意事项 1>
  - <注意事项 2>

══════════════════════════════════════════════
```

## 二、各字段生成规则

### 2.1 建议发布日期

| 内容类型 | 建议日期 | 说明 |
|---------|---------|------|
| `daily-news` | 内容日期当天 | 日报时效性最强 |
| `hotspot-analysis` (weekly) | 内容日期所在周的周一或周日 | 周一适合"本周展望"，周日适合"本周回顾" |
| `hotspot-analysis` (monthly) | 次月 1-3 日 | 月初发布月度回顾 |
| `github-project` | 周三或周四 | 工作日中段，开发者活跃 |
| `topic-deep-dive` | 周二或周三 | 避开周一繁忙和周五松懈 |
| `product-recommendation` | 周四或周五 | 读者周末前有探索新工具的时间 |
| `social-insight` | 周二或周四 | 深度内容适合工作日阅读 |
| `content-advice` | 不限 | 选题建议不公开发布 |

### 2.2 建议发布时间

| 时段 | 时间 | 适用场景 |
|------|------|---------|
| 早间通勤 | 7:30-8:30 | 日报 |
| 上午工作间隙 | 10:00-11:00 | 技术深度内容 |
| 午休 | 12:00-13:00 | 日报、轻量推荐 |
| 下午茶 | 15:00-16:00 | 一般性内容 |
| 晚间阅读 | 20:00-22:00 | 深度专题、周报、月报 |

**原则：** 基于微信公众号阅读高峰（早 8-9、午 12-13、晚 20-22），结合内容类型的阅读深度。

### 2.3 标题变体

为每个内容生成 4 个标题变体：

| 变体 | 模式 | 示例 |
|------|------|------|
| A. 主标题 | 标准陈述式 | "2026年6月第三周 AI 热点回顾" |
| B. 疑问句 | 以问句激发好奇心 | "这周 AI 圈发生了什么？OpenAI/Meta/国内厂商最新动态一文看完" |
| C. 数字驱动 | 用数字制造期待 | "8 条热点 + 2 大趋势：本周 AI 圈精华一览" |
| D. 悬念式 | 留白+引导点击 | "OpenAI 三连发、Meta 开源杀器...这周的 AI 圈太疯狂了" |

**原则：** 所有变体必须在事实范围内，不标题党、不夸大。C 和 D 变体仅用于社交媒体转发语，正式发布仍用 A。

### 2.4 封面方向

基于内容主题给出视觉建议：

```
🖼️ 封面方向：
  - 主视觉建议：<描述>（如"OpenAI GPT-5o 的官方主视觉截图"）
  - 配色建议：<描述>（如"以 OpenAI 品牌色（黑+绿）为主调"）
  - 排版方向：<大标题居中 / 左对齐摘要 / 图文混排>
  - 生成工具：<建议使用 infographic-carousel / Midjourney / 官方素材>
```

## 三、传播建议

### 3.1 转发语

为微信公众号生成 1-2 句转发语（适合分享到朋友圈/微信群时粘贴）。

```
📊 转发语建议：
"本周 AI 圈继续卷多模态：OpenAI 发了三款音频模型，Meta 开源了 VLM-3，
国内厂商也在跟进。一文速览本周 8 条核心热点 + 2 大趋势信号。"
```

**原则：** ≤140 字（方便复制），含核心信息点，不用"必看""震惊"等营销词。

### 3.2 社群分享角度

根据内容类型推荐在什么社群、用什么角度分享：

```
📱 社群分享建议：
  • 技术群 → 侧重："Meta VLM-3 的技术细节和开源协议"
  • 产品群 → 侧重："OpenAI 音频模型的产品定位和定价策略"
  • 投资群 → 侧重："多模态赛道投融资动态和竞争格局"
```

## 四、注意事项

根据内容的具体情况生成注意事项：

常见事项：
- "文中图片使用了外部链接，发布前需上传至微信公众号素材库"
- "文末建议添加关注引导和往期回顾链接"
- "转发语中的 emoji 在部分 Android 机型可能显示异常，建议去掉或用文字替代"
- "文中的对比表格在微信中宽度可能超出，建议在手机上预览确认"
- "原创声明：文中内容如引用外部数据，建议在文末声明出处"

## 五、简化版（快速发布用）

当用户不需要详细建议时，输出精简版：

```
📋 发布建议（精简）
  日期：<YYYY-MM-DD> <星期X> <时间>
  标题：<主标题>
  封面：<一句话建议>
  注意：<核心注意点>
```

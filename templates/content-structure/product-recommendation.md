## 二、各章节详解

### 2.1 标题

**格式：** `推荐 N 款 <描述> 的 <品类> 工具/产品` 或 `<主题> 产品推荐`

**示例：**
- `2026年6月 AI 视频生成工具推荐`
- `推荐 5 款本地大模型推理工具：从免费到企业级`
- `AI 编程助手横评：这 4 款工具各有什么优劣`

**原则：** 标题应体现品类和推荐数量，让读者一眼知道"推荐什么"和"几款可选"。

---

### 2.2 一、推荐概览（200-400 字）

**目的：** 说明本期推荐的背景和选题理由。

**内容块：**

1. **选题理由**：为什么推荐这类产品？（用户需求/行业趋势/工具生态变化）
2. **筛选说明**：简要说明筛选标准（功能/价格/体验/适用人群）。
3. **分组预览**：一句话预告本期包含哪几个分组，每组面向什么人群。

**示例：**

```markdown
## 一、推荐概览

AI 视频生成工具在 2026 年上半年经历了爆发式增长——从 3 秒 GIF 到 2 分钟叙事短片，
能力边界不断被突破。但面对十几款产品，选哪个适合自己的需求？

本期从功能完整性、价格合理性和上手难度三个维度筛选了 6 款产品，
按使用场景分为两组：

- 🎬 专业创作者（3 款）：需要精确控制、高质量输出
- 🚀 快速尝鲜（3 款）：低门槛、快速出片、适合社交媒体
```

---

### 2.3 二、产品分组（核心章节，1000-2000 字）

**目的：** 按用户画像或使用场景分组呈现产品，每组 2-5 个产品。

#### 分组标题

按"谁适合用"或"什么场景用"命名，体现分组的实用价值：

```markdown
### 🎬 专业创作者
### 🚀 快速尝鲜
### 💰 免费开源方案
### 🏢 企业/团队协作
### 🔧 开发者/API 优先
```

**分组维度选择原则：** 选最能帮助读者"对号入座"的维度。优先使用场景，其次用户画像，再次价格层级。

#### 产品卡片（每个产品 200-350 字）

```markdown
#### <产品名>

**一句话定位**：<产品简要介绍，1 句话说清"这是做什么的">

**核心功能：**
- <功能 1>
- <功能 2>
- <功能 3>
- <功能 4>（可选）
- <功能 5>（可选）

| 属性 | 值 |
|------|-----|
| 💰 价格 | <免费/付费/分层定价，标注来源日期> |
| 👥 适用人群 | <谁适合用这个产品> |
| 🆚 主要竞品 | <同类替代品名称> |

**编辑点评：** <编辑视角：推荐理由/已知局限/注意事项> [^ref-xxx]
```

**产品卡片约束：**
- 价格必须真实，标注来源日期（"截至 2026-06，官网定价"）。
- 核心功能 3-5 点，每点一句话，不堆砌术语。
- 适用人群一句话说清，不用"适合所有人"。
- 竞品对比指出差异化优势，不只写名字。

### 2.4 三、对比总览（200-400 字，推荐使用）

**目的：** 当同组 ≥2 个产品时用对比表帮助读者快速决策。

**呈现方式：**

```markdown
## 三、对比总览

| 产品 | 定位 | 价格 | 核心优势 | 主要短板 | 最适合 |
|------|------|------|---------|---------|--------|
| Runway | 专业视频生成 | $15/月起 | 控制精度最高 | 价格高 | 专业创作者 |
| Pika | 创意快速出片 | 免费/ $10月 | 上手最快 | 时长受限 | 社交媒体 |
| MoonValley | 叙事短片 | $12/月起 | 叙事连贯性好 | 风格受限 | 内容创作者 |
```

**使用条件：** 推荐同组 ≥2 个产品时使用。不强行对比性质完全不同的产品。

---

### 2.5 四、选择建议（200-400 字）

**目的：** 帮助不同需求的读者快速找到适合自己的产品。

**呈现方式：**

```markdown
## 四、选择建议

- **预算有限、想快速上手？** → Pika，免费层足够体验核心功能。
- **需要专业级视频制作？** → Runway，控制精度和输出质量最高。
- **做 AI 短视频自媒体？** → MoonValley，叙事连贯性最适合讲故事。
- **团队协作、需要版本管理？** → Runway Teams 计划，支持多人协作和资产库。
```

**原则：**
- 基于产品实际特性给出建议，不主观偏好。
- 如果某个产品有明显缺陷，在建议中诚实说明。
- 不同建议之间避免矛盾（如"A 最好但价格高 → 选 B，但 B 功能少 → 选 C"这样的死循环）。

## 三、分组逻辑指南

### 3.1 分组维度选择

| 维度 | 适用场景 | 示例 |
|------|---------|------|
| 使用场景 | 产品功能差异大 | 专业创作 / 快速尝鲜 / API 对接 |
| 用户画像 | 面向不同技能水平的用户 | 新手友好 / 专业用户 / 企业团队 |
| 价格层级 | 预算差异显著 | 免费开源 / 个人付费 / 企业级 |
| 部署方式 | 云端 vs 本地差异大 | 云端 SaaS / 本地部署 / 混合 |

**选择原则：** 选最能帮助读者"对号入座"的维度。优先使用场景。

### 3.2 分组数量

- 最少：1 组（如品类非常聚焦）
- 推荐：2-3 组
- 最多：4 组
- 每组：2-5 个产品

## 四、产品卡片的可选扩展

以下为可选字段，按需使用：

| 扩展字段 | 用途 | 使用条件 |
|---------|------|---------|
| 上手难度 | 🟢 简单 / 🟡 中等 / 🔴 复杂 | 产品间上手难度差异大 |
| 平台支持 | Web / macOS / Windows / iOS / Android | 多平台支持是差异化因素 |
| 已知局限 | 功能、性能、兼容性等问题 | 编辑在调研中发现明显短板 |
| 替代建议 | "如果 XX 不适合你，试 YY" | 同组产品有明确的替代关系 |
| 试用建议 | 免费层额度、试用期、退款政策 | 影响用户决策的试用信息 |

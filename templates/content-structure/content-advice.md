# 推荐内容建议 — 结构指南

> 路径：templates/content-structure/content-advice.md
> 职责：定义推荐内容建议（content-advice）的建议卡片格式和输出结构
> Phase：2
> 依赖：references/content-types-guide.md（content-advice 定义）
> 被依赖：workflows/content-advice.md
> 最后更新：2026-06-17

---

## 一、核心定位

内容建议是一个**轻量信息扫描工具**，不是完整的内容生产流程。

- **输入**：当日/近期热点的快速扫描结果
- **输出**：结构化的建议卡片文档（非 content-package）
- **用途**：为编辑提供选题灵感、标题方向和传播角度参考
- **后续**：用户从建议中挑选的选题 → 进入对应 workflow（daily-news / topic-deep-dive / hotspot-analysis）从 L1 开始完整生产

---

## 二、输出格式

### 2.1 整体结构

```markdown
# 内容选题建议 | <日期>

## 一、扫描概况
## 二、选题建议卡片（N 个）
## 三、快速排除
```

### 2.2 一、扫描概况（50-100 字）

一句话总结扫描范围和发现。

**示例：**

```markdown
## 一、扫描概况

今日扫描 28 条 AI/科技热点，从中筛选出 4 个有内容价值的选题方向。
覆盖平台：知乎热榜、微博热搜、36Kr、机器之心。
```

---

### 2.3 二、选题建议卡片

每个选题一张卡片，使用以下结构：

```markdown
## 📌 选题 N：<一句话选题概述>

| 维度 | 内容 |
|------|------|
| **选题概述** | <一句话描述选题内容，让读者立刻理解这个选题在讲什么> |
| **信号来源** | <支撑这个选题的热点/趋势/事件，2-3 条> |
| **推荐理由** | <为什么值得写：时效性/用户兴趣/行业重要性> |
| **标题方向** | ① <标题 1>（风格：资讯型/分析型/观点型）<br>② <标题 2>（风格：…）<br>③ <标题 3>（风格：…） |
| **传播角度** | <适合什么平台/读者群/时机> |
| **建议类型** | <daily-news / topic-deep-dive / hotspot-analysis / github-project> |
| **优先级** | 🔴 高 / 🟡 中 / 🟢 低 |

---
```

#### 字段说明

| 字段 | 说明 | 是否必需 |
|------|------|---------|
| 选题概述 | 一句话让编辑理解选题内容 | ✅ |
| 信号来源 | 支撑选题的热点信号（不需要完整验证，给大致方向即可） | ✅ |
| 推荐理由 | 为什么值得写（最多 2 句） | ✅ |
| 标题方向 | 2-3 个可能标题，标注风格类型 | ✅ |
| 传播角度 | 适合什么平台/读者/时机 | ❌ 可选 |
| 建议类型 | 推荐进入哪个 workflow | ✅ |
| 优先级 | 🔴高 / 🟡中 / 🟢低 | ✅ |

---

### 2.4 三、快速排除（可选）

简要说明被扫描但未入选选题的热点，及排除理由。

**示例：**

```markdown
## 三、快速排除

- **XX 公司融资新闻**：纯商业信息，无技术内容，不适合本刊定位。
- **YY 八卦/花边**：与 AI/科技无关。
- **ZZ 产品促销**：广告性质，无编辑价值。
```

## 三、完整示例

```markdown
# 内容选题建议 | 2026年6月17日

## 一、扫描概况

今日扫描 28 条 AI/科技热点，覆盖知乎、微博、36Kr、机器之心、Hacker News。
筛选出 3 个有内容价值的选题方向。

---

## 📌 选题 1：OpenAI 定价策略大调整，API 经济进入新阶段

| 维度 | 内容 |
|------|------|
| **选题概述** | OpenAI 今日大幅调整 GPT-5o 系列 API 定价，mini 降 40%，speech 改按字符计费。这是对开源模型竞争和客户成本压力的直接回应。 |
| **信号来源** | ① OpenAI 官方博客公告（6月17日）② Hacker News 热榜第一（600+ 评论）③ 多家科技媒体跟进报道 |
| **推荐理由** | 直接影响所有 AI 应用开发者的成本结构。'API 价格战'是当前行业核心话题之一。 |
| **标题方向** | ① OpenAI 降价 40%，API 价格战进入白热化（资讯型）<br>② API 降价潮下，AI 应用的商业模式如何重构？（分析型）<br>③ 从 OpenAI 降价看开源模型的'鲶鱼效应'（观点型） |
| **传播角度** | 微信公众号深度分析、知乎问答。目标读者：AI 开发者、创业者。时效窗口：24-48h。 |
| **建议类型** | topic-deep-dive（API 经济专题） |
| **优先级** | 🔴 高 |

---

## 📌 选题 2：今日 AI 要闻速览

| 维度 | 内容 |
|------|------|
| **选题概述** | 今日有多条重要但无需深度的新闻：OpenAI 降价、Meta 开源 VLM、字节视频工具。适合用日报格式覆盖。 |
| **信号来源** | 今日 28 条热榜新闻中筛选 6 条核心要闻 |
| **推荐理由** | 每日固定栏目，信息量大但单条深度要求低。 |
| **标题方向** | ① AI 科技日报 | 6月17日（资讯型）<br>② 今日 AI：OpenAI 降价、Meta 开源、字节入局视频（资讯型） |
| **传播角度** | 微信公众号早晨推送、飞书群每日播报。 |
| **建议类型** | daily-news |
| **优先级** | 🔴 高 |

---

## 📌 选题 3：AI 视频编辑工具爆发，创作者经济新变量

| 维度 | 内容 |
|------|------|
| **选题概述** | 字节"即梦 AI"发布，加上此前 Runway、Pika 的更新，AI 视频编辑工具赛道快速升温。 |
| **信号来源** | ① 字节"即梦 AI"公测（6月17日）② Runway Gen-4 上周更新 ③ 知乎"AI视频工具"话题上升 |
| **推荐理由** | 赛道趋势明显，多个产品集中发布，有产业分析价值。 |
| **标题方向** | ① AI 视频编辑赛道爆发：Runway vs Pika vs 即梦（对比型）<br>② 从"拍视频"到"写视频"：AI 如何改变创作者工作流（分析型） |
| **传播角度** | 微信公众号、B站专栏。目标读者：创作者、内容团队。时效不急（可打磨 2-3 天）。 |
| **建议类型** | topic-deep-dive（产业分析） |
| **优先级** | 🟡 中 |

---

## 三、快速排除

- **某网红AI产品营销稿**：纯软文，无信息增量。
- **某公司C轮融资**：金额不大，无技术创新点。
- **某技术博客水文**：标题党，内容无实质信息。
```

## 四、与完整 content-package 的差异

| 维度 | 内容建议 | content-package |
|------|---------|----------------|
| 内容 | 选题池（多个选题建议） | 单篇完整内容 |
| 深度 | 信号级别 | 成品级别 |
| 来源验证 | 不需要三项硬检查 | 必须通过三项硬检查 |
| 格式 | Markdown 建议文档 | manifest.json + body.md + assets/ |
| 确认流程 | 1 层（选题确认，可选） | 3 层（信息→内容→格式） |
| 后续流转 | 用户挑选→进入 workflow | 完成后→进入 L3 格式消费 |
| 存储 | `outputs/content-advice-<date>.md` | `outputs/<type>-<date>/` 目录 |

---

## 五、约束

- ❌ 不生成 manifest.json
- ❌ 不生成 body.md（建议文档本身就是最终产出）
- ❌ 不需要完整来源验证（信号级别不要求 link_accessible/source_verified）
- ❌ 不进入格式消费（L3）
- ✅ 每条建议至少包含：选题概述 + 推荐理由 + 建议类型 + 优先级
- ✅ 建议数量：3-8 个（太少无选择余地，太多无重点）
- ✅ 排重：同一方向的不同角度合并为一条建议

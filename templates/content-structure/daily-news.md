## 一、结构概览

```
# <标题>
## 一、今日速览
## 二、要闻列表（5-10 条）
## 三、明日关注
## 参考来源
```

---

## 二、各章节详解

### 2.1 标题

**格式：** `AI/科技日报 | <日期>` 或 `<日期> AI 科技要闻`

**示例：**
- `AI 科技日报 | 2026年6月17日`
- `6月17日 AI 要闻速览`
- `今日 AI：OpenAI 更新定价策略、Meta 开源新模型、字节发布视觉工具`

**原则：** 日期必须准确。如有特别突出的当日事件，可在标题中体现，否则用标准日期格式。

---

### 2.2 一、今日速览（100-200 字）

**目的：** 一句话覆盖今日最重要的 2-3 件事，让读者 10 秒内了解今天发生了什么。

**内容块：**

1. **日期标注**：一句话标注日期和星期。
2. **核心事件**：今日最重要的 1-3 件事（一句话概括每件）。
3. **数据概览**：可选，如"今日共监测 XX 条 AI/科技新闻"。

**示例：**

```markdown
## 一、今日速览

2026年6月17日，星期三。OpenAI 调整 API 定价策略，Meta 开源新一代视觉模型，
字节跳动发布 AI 视频编辑工具。今日共监测 28 条 AI/科技要闻。

🏷️ 今日关键词：API定价 · 视觉模型 · AI视频 · 开源
```

**约束：**
- ❌ 不展开分析（展开留给第二节）
- ❌ 不输出编辑视角/趋势信号
- ✅ 控制在 3 句话以内

---

### 2.3 二、要闻列表（核心章节，400-1000 字）

**目的：** 按重要性排序，逐条呈现今日要闻。每条简讯式，2-3 句话 + 来源引用。

**数量：** 5-10 条。多于 10 条时精简，少于 5 条时检查数据源覆盖。

**排序规则：**
- 按编辑判断的重要性降序（非评分）。
- 同类事件可合并为一条（如"多家厂商跟进 XX"）。
- 同源的多条新闻如果关联性强可合并。

**每条结构：**

```markdown
### <序号>. <事件标题>

<事件概述：2-3 句话。包含：发生了什么 + 关键参与者 + 核心事实。>

> 📎 来源：[^ref-xxx]（平台/发布日期）
```

**示例：**

```markdown
### 1. OpenAI 调整 API 定价策略

OpenAI 宣布即日起调整 GPT-5o 系列 API 定价，其中 GPT-5o-mini 价格下调 40%，
GPT-5o-speech 新增按字符计费模式。这一调整被认为是对近期开源模型竞争加剧的回应。

> 📎 来源：[^ref-001] OpenAI 官方博客（6月17日）

### 2. Meta 开源 VLM-3.1 视觉模型

Meta 发布 VLM-3.1 系列更新，在文档理解基准上比前代提升 15%。
模型采用 Apache 2.0 许可证，已在 Hugging Face 上架。

> 📎 来源：[^ref-002] Meta AI Blog（6月17日）

### 3. 字节跳动发布 AI 视频编辑工具

字节跳动正式对外发布"即梦 AI"视频编辑工具，支持文本生成视频片段、
智能剪辑和风格迁移，目前面向创作者开放公测。

> 📎 来源：[^ref-003] 字节跳动官方公告（6月17日）
```

**日报条目约束：**

- ❌ 不展开深度分析（留给热点分析/通用专题）
- ❌ 不含编辑视角信号块（💡📈👀⚠️ — 这些属于热点分析）
- ❌ 不含趋势判断（"这意味着……""长期来看……"）
- ✅ 每条 2-3 句话即可：事实陈述优先
- ✅ 来源引用必须包含发布日期（确认在 24h 内）
- ✅ 如果有特别重要的编辑说明（如"此事件可能持续发酵，明天关注"），可极简标注，不展开

**分组建议：**

当要闻超过 8 条时，可按主题分组呈现（不强制）：

```markdown
### 🔥 重要发布（3 条）
### 📊 行业动态（3 条）  
### 🔧 工具与开源（2 条）
### 📋 政策与监管（1 条）
```

### 2.4 三、明日关注（100-200 字）

**目的：** 提醒读者明天可能发生的重要事件。不必每天都有内容。

**内容块：**

1. **已知事件**：明天的产品发布、会议、截止日期等。
2. **延续关注**：今天的热点事件在明天可能的新进展。
3. **编辑提示**：今天被略过但明天可能变得重要的事件。

**示例：**

```markdown
## 三、明日关注

- **Google I/O 预热活动**（6月18日）：预计有 Gemini 新版本更新。
- **OpenAI 定价调整影响**：关注开发者社区反馈和竞品定价反应。
- **字节"即梦 AI"公测数据**：明日可能公布首日用户数据。
```

**原则：**
- 只写有信息来源的展望，不凭空预测。
- "可能""值得关注""预计"等谨慎措辞。
- 无内容时可省略本章节。不强行为填满而编造。

---

### 2.5 参考来源

**格式：** 自动从 manifest.json 的 references[] 生成。

```markdown

[^ref-003]: 字节跳动 — "即梦 AI 正式开放公测"（2026-06-17）
...
```

## 三、日报与热点分析的关键差异

| 维度 | 日报 | 热点分析 |
|------|------|---------|
| 时间窗口 | 24h | 7-30 天 |
| 条目深度 | 简讯（2-3 句/条） | 展开（100-200 字/条） |
| 信号块 | 无（💡📈👀⚠️ 均不用） | 有 |
| 趋势分析 | 无 | 有 |
| 周期对比 | 无 | 有 |
| 展望 | 仅"明日关注"（选项） | "下周/下月展望"（常规） |
| 叙事线 | 无（纯列表式） | 有（贯通全文） |
| 编辑视角 | 极简，仅必要时标注 | 常规板块 |

## 四、日报专属：时效门槛规则

日报的唯一硬门槛是时效性。

### 4.1 时效门槛

```
item.published_at 必须在 [当前时间 - 24h, 当前时间] 范围内
```

### 4.2 特殊情况处理

| 情况 | 处理方式 |
|------|---------|
| 事件跨越多日（如持续 3 天的发布会） | 保留，报道当日新进展。editorial_note 注明"Day N of M" |
| 昨日深夜发布（如 23:00） | 纳入今日日报（仍在 24h 窗口内），标注"昨夜" |
| 无明确发布时间 | 标记 `published_at` 为空，editorial_note 说明。不自动排除但降级 |
| 重要但超过 24h 的事件 | 不纳入日报。若确需报道 → 建议用户考虑热点分析或通用专题 |

### 4.3 数据不足时的降级

当日候选少于 5 条时：
1. 检查 data-sources.md 降级链 → 启用下一优先级工具补充
2. 放宽平台范围（扩大到非主流来源）
3. 仍不足 → 在 editorial_notes 中声明"今日数据覆盖不足"，如实输出

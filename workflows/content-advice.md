# 推荐内容建议 — 工作流

> 路径：workflows/content-advice.md
> 职责：定义推荐内容建议（content-advice）的轻量扫描流程，从信号扫描到建议文档生成
> Phase：2
> 依赖：CONSTITUTION.md, references/data-sources.md, references/content-types-guide.md（第八节）, templates/content-structure/content-advice.md
> 生产模式：轻量扫描（信号扫描 → 选题生成 → 建议卡片输出）
> 最后更新：2026-06-17

---

## 资源加载清单

Agent 激活本 workflow 时，按 SKILL.md 定义的顺序加载：

| 阶段 | 加载资源 |
|------|---------|
| 启动 | `CONSTITUTION.md` |
| 扫描 | `references/data-sources.md`（trending_discovery 降级链）→ `references/content-types-guide.md`（第八节） |
| 输出 | `templates/content-structure/content-advice.md` |

> **不加载：** `references/source-and-verification.md`（不需要完整来源验证）、`references/confirmation/`（确认可选）。内容建议是轻量流程，不进入完整的 L1→L2→L3→L4 管线。

## 能力声明

本 workflow 需要以下数据能力（声明能力名，不声明工具名）：

| 能力 | 说明 | 门槛 |
|------|------|------|
| `trending_discovery` | 当日/近期热点快速扫描，获取新闻标题和基本摘要 | ≥ 10 条候选信号 |

> **明确不需要的能力：** `web_verification`（信号级别不需完整验证）、`web_search`（深度检索留给后续 workflow）、`trend_signal`、`period_comparison`、评分。内容建议的目标是快速发现选题，不是生产完整内容。

---

## 轻量扫描流程

内容建议**不进入**完整的 L1→L2→L3 管线。它有自己的简化流程：

```
信号扫描
    │
    ▼
选题识别（编辑快速判断）
    │
    ▼
建议卡片生成
    │
    ▼
输出建议文档
    │
    ▼
[可选] 选题确认（仅 1 层，用户可跳过）
    │
    ▼
用户挑选选题 → 进入对应 workflow 从 L1 开始完整生产
```

---

## 步骤 1：信号扫描

```
能力：trending_discovery

操作：
1. 按 data-sources.md → trending_discovery 降级链获取当日/近期热点
2. 扫描门槛：≥ 10 条候选信号
3. 不满足门槛 → 扩大时间范围（±12h）或增加平台
4. 快速浏览标题和摘要，不需要打开原文

输出：热点信号列表（标题 + 摘要 + 平台 + 日期）
```

**与日报/热点分析的关键区别：** 此阶段只看标题和摘要，不打开原文、不验证来源、不判断时效。目的是快速筛选"可能值得写的方向"。

---

## 步骤 2：选题识别（编辑快速判断）

从信号列表中识别有内容价值的选题方向。

### 判断维度

| 维度 | 问题 | 判断方式 |
|------|------|---------|
| 选题潜力 | 这件事能否支撑一篇独立内容？ | 信号如果只是"一条新闻"→ 不适合做专题。信号如果是"一个趋势的体现"→ 适合做专题。 |
| 传播性 | 目标读者会不会关心？ | 与 AI/科技从业者的日常相关度。"XX 公司发了个新版本"不一定重要，"API 价格战对开发者意味着什么"更重要。 |
| 信息增量 | 除了复述新闻，还能提供什么？ | 编辑分析、背景补充、横向对比、趋势提炼。如果只能复述 → 放日报。 |
| 时效窗口 | 是否有时效压力？ | 热点类选题 24-48h 窗口。分析类选题可打磨 2-3 天。 |

### 选题与内容类型的映射

基于判断结果，为每个选题推荐合适的内容类型：

```
信号是"一件事" + 时效强 → daily-news
信号是"一个趋势" + 有数据 → hotspot-analysis
信号是"一个话题" + 可深度分析 → topic-deep-dive
信号是"一个好项目" → github-project
信号是"多个方向的信号" → 拆分为多条建议
```

### 排重规则

同一方向的不同角度合并为一条建议。例如：
- "OpenAI 降价" + "多家厂商跟进降价" → 合并为"API 价格战"专题建议
- 不拆分为"OpenAI 降价"和"竞品反应"两条独立建议

## 步骤 3：生成建议卡片

对每个识别出的选题，按 `templates/content-structure/content-advice.md` 的卡片格式生成建议：

```markdown
## 📌 选题 N：<一句话选题概述>

| 维度 | 内容 |
|------|------|
| **选题概述** | ... |
| **信号来源** | ... |
| **推荐理由** | ... |
| **标题方向** | ① ... ② ... ③ ... |
| **传播角度** | ... |
| **建议类型** | daily-news / topic-deep-dive / ... |
| **优先级** | 🔴 高 / 🟡 中 / 🟢 低 |

---
```

### 标题方向风格标注

每个标题方向标注风格类型，帮助编辑理解标题的定位：

| 风格 | 特点 | 适用场景 |
|------|------|---------|
| 资讯型 | 事实陈述，不含观点 | 日报、快讯 |
| 分析型 | 有分析视角，回答"为什么/意味着什么" | 专题、深度分析 |
| 观点型 | 有编辑立场，回答"好还是不好/该怎么做" | 评论、观点文章 |
| 对比型 | 横向对比多个对象 | 产品推荐、产业分析 |
| 科普型 | 解释技术概念，降低理解门槛 | 技术分析 |

### 优先级标准

| 优先级 | 标准 |
|--------|------|
| 🔴 高 | 时效强 + 传播性高 + 信息增量大。建议当天/次日启动。 |
| 🟡 中 | 有价值但时效不急，或需要更多素材准备。可安排在本周。 |
| 🟢 低 | 可写但非优先。可放入选题池备用。 |

### 建议数量控制

- 推荐：3-5 条
- 最少：1 条（当日信号确实少）
- 最多：8 条（超过 8 条说明编辑判断不够聚焦，精简）

## 步骤 4：输出建议文档

```
输出路径：outputs/content-advice-<YYYY-MM-DD>.md

不是 content-package 目录结构。是单个 Markdown 文件。
```

### 输出格式

```markdown
# 内容选题建议 | <日期>

## 一、扫描概况
...

## 二、选题建议卡片
[卡片 1]
[卡片 2]
...

## 三、快速排除
...
```

完整格式参考 `templates/content-structure/content-advice.md`。

---

## 步骤 5：[可选] 选题确认

```
呈现：
1. 扫描概况 — 扫描了多少信号、平台覆盖
2. 选题卡片预览 — 所有建议卡片的标题和优先级
3. 编辑选择 — "你想深入哪个选题？或者我全量展开哪个？"

注：内容建议的确认是可选且轻量的。用户可以直接挑选选题进入生产。
用户也可以说"全写出来"→ Agent 展开所有卡片。
用户也可以说"不用确认，直接出"→ Agent 输出完整建议文档。
```

## 后续：从建议到生产

用户从建议中挑选选题后，Agent 进入对应 workflow 从 L1 开始：

```
用户："展开选题 1"
  → Agent 加载 workflows/topic-deep-dive.md
  → 从 L1 步骤 1 开始（选题边界已在建议中明确，可直接进入步骤 2）

用户："出日报"
  → Agent 加载 workflows/daily-news.md
  → 从 L1 步骤 1 开始完整流程
```

## 内容建议的差异特征

| 维度 | 内容建议 | 日报 | 通用专题 |
|------|---------|------|---------|
| 流程 | 轻量扫描（4 步） | 完整流程（8 步+L3） | 完整流程（9 步+L3） |
| 输出 | 单文件 Markdown | 目录（manifest+body+assets） | 目录（manifest+body+assets） |
| 来源验证 | 不需要 | 需要 | 需要 |
| 确认 | 1 层（可选） | 2 层 | 2 层 |
| 确认协议 | 不加载 | 加载 | 加载 |
| 后续 | 用户挑选→进入 workflow | 完成后→L3 格式消费 | 完成后→L3 格式消费 |
| 时效要求 | 越快越好（选题有时效窗口） | 24h 内完成 | 不着急（打磨品质优先） |

## 约束

- ❌ **不产出 content-package**（不生成 manifest.json + body.md + assets/）
- ❌ 不加载 `references/source-and-verification.md`（不需要三项硬检查）
- ❌ 不加载 `references/confirmation/`（确认可选）
- ❌ 不进入 L3 格式消费
- ✅ 输出为单个 Markdown 文件：`outputs/content-advice-<date>.md`
- ✅ 建议卡片至少包含：选题概述、推荐理由、建议类型、优先级
- ✅ 标题方向至少 2 个，每个标注风格
- ✅ 排重：同方向的不同角度合并
- ✅ 建议数量控制在 3-8 条
